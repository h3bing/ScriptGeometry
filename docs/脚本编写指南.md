# 脚本编写指南

本文档介绍如何为ScriptGeometry编写TOON格式的几何生成脚本。

## TOON格式概述

TOON (Text Object-Oriented Notation) 是ScriptGeometry使用的脚本格式，包含三个部分：

```
[meta]     - 元数据（脚本标识、名称、类别等）
[attrs]    - 属性定义（用户可调整的参数）
[code]     - C代码（几何生成逻辑）
```

## 完整示例

```toon
[meta]
id          = "custom_gear"
name        = "自定义齿轮"
category    = "solid"
version     = "1.0"
author      = "Your Name"
description = "可配置齿数和尺寸的齿轮"

[attrs]
teeth        int     20      "齿数"
outerRadius  float   2.0     "外半径"
innerRadius  float   0.5     "内半径"
thickness    float   0.3     "厚度"
toothDepth   float   0.2     "齿深"

[code]
void build(Entity entity, GeoData geo) {
    int teeth = entity_getInt(entity, "teeth");
    float outer = entity_getFloat(entity, "outerRadius");
    float inner = entity_getFloat(entity, "innerRadius");
    float thick = entity_getFloat(entity, "thickness");
    float depth = entity_getFloat(entity, "toothDepth");
    
    solid_factory_gear(geo, 0.0f, 0.0f, 0.0f, teeth, outer, inner, thick, depth);
}
```

## [meta] 元数据

| 字段 | 必需 | 说明 |
|------|------|------|
| `id` | ✅ | 唯一标识符，用于脚本注册和查找 |
| `name` | ✅ | 显示名称，出现在UI中 |
| `category` | | 类别：`solid`/`surface`/`curve` |
| `version` | | 版本号，默认"1.0" |
| `author` | | 作者信息 |
| `description` | | 功能描述 |

## [attrs] 属性定义

格式：`名称 类型 默认值 "标签" "描述"`

### 支持的类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `float` | 浮点数 | `radius float 1.0 "半径"` |
| `int` | 整数 | `segments int 32 "分段数"` |
| `bool` | 布尔值 | `closed bool true "是否闭合"` |
| `string` | 字符串 | `name string "Box" "名称"` |
| `color` | 颜色 | `color color "1,0,0,1" "颜色"` |
| `enum` | 枚举 | `style enum 0 "样式"` |

### 示例

```toon
[attrs]
radius     float   1.0     "半径"      "球体的半径"
segments   int     32      "分段数"    "经度方向分段"
rings      int     16      "环数"      "纬度方向分段"
wireframe  bool    false   "线框模式"  "是否显示线框"
mainColor  color   "0.2,0.5,0.8,1"  "主色调"
```

## 编写技巧

### 1. 参数验证

```c
void build(Entity entity, GeoData geo) {
    float radius = entity_getFloat(entity, "radius");
    int segments = entity_getInt(entity, "segments");
    
    // 参数约束
    if (radius <= 0.0f) radius = 1.0f;
    if (segments < 3) segments = 3;
    if (segments > 256) segments = 256;
    
    solid_factory_sphere(geo, 0, 0, 0, radius, segments/2, segments);
}
```

### 2. 使用数学函数

```c

    float angle = entity_getFloat(entity, "angle");  // 度数
    float radians = math_deg2rad(angle);
    
    float x = math_cos(radians);
    float y = math_sin(radians);
    
    // 创建旋转后的几何体...
}
```

### 3. 组合多个形状

```c
void build(Entity entity, GeoData geo) {
    float radius = entity_getFloat(entity, "radius");
    float height = entity_getFloat(entity, "height");
    
    // 底部球体
    solid_factory_sphere(geo, 0, 0, 0, radius, 16, 32);
    
    // 顶部圆柱
    solid_factory_cylinder(geo, 0, radius + height/2, 0, radius, height, 32, 1);
}
```

### 4. 条件分支

```c
void build(Entity entity, GeoData geo) {
    int shapeType = entity_getInt(entity, "shapeType");
    float size = entity_getFloat(entity, "size");
    
    switch (shapeType) {
        case 0:
            solid_factory_box(geo, 0, 0, 0, size, size, size);
            break;
        case 1:
            solid_factory_sphere(geo, 0, 0, 0, size/2, 16, 32);
            break;
        case 2:
            solid_factory_cylinder(geo, 0, 0, 0, size/2, size, 32, 1);
            break;
    }
}
```

## 调试技巧

### 输出调试信息

由于TCC脚本运行在沙箱环境中，无法使用printf。可以通过创建特殊的调试实体来查看值：

```c
// 在脚本中使用固定值确保几何生成
solid_factory_sphere(geo, 0, 0, 0, 1.0f, 16, 32);  // 测试基础功能
```

### 常见错误

1. **空指针访问**: 始终检查返回值
2. **数组越界**: 注意顶点索引范围
3. **除零错误**: 验证除数不为零
4. **类型错误**: 确保使用正确的entity_get函数

## 完整示例：参数化螺旋楼梯

```toon
[meta]
id          = "spiral_stairs"
name        = "螺旋楼梯"
category    = "solid"
version     = "1.0"
description = "参数化螺旋楼梯生成器"

[attrs]
radius       float   2.0    "半径"
height       float   5.0    "总高度"
turns        float   2.0    "圈数"
steps        int     20     "台阶数"
stepHeight   float   0.15   "台阶厚度"

[code]
void build(Entity entity, GeoData geo) {
    float radius = entity_getFloat(entity, "radius");
    float height = entity_getFloat(entity, "height");
    float turns = entity_getFloat(entity, "turns");
    int steps = entity_getInt(entity, "steps");
    float stepH = entity_getFloat(entity, "stepHeight");
    
    // 参数约束
    if (steps < 1) steps = 1;
    if (radius <= 0) radius = 1.0f;
    
    for (int i = 0; i < steps; i++) {
        float t = (float)i / steps;
        float angle = turns * 2 * CAPI_PI * t;
        float y = height * t;
        float x = radius * math_cos(angle);
        float z = radius * math_sin(angle);
        
        // 创建台阶
        solid_factory_box(geo, x, y, z, 0.8f, stepH, 0.4f);
    }
    
    // 中心柱
    solid_factory_cylinder(geo, 0, 0, 0, 0.1f, height, 16, 1);
}
```

## 最佳实践

1. **命名规范**: 使用有意义的属性名
2. **合理默认值**: 提供合理的默认参数
3. **参数范围**: 在代码中验证参数范围
4. **性能考虑**: 避免过高的细分数量
5. **文档注释**: 为属性添加描述

## 常见问题

**Q: 脚本修改后如何生效？**

A: 修改脚本文件后，脚本库会自动检测变化。也可以在UI中重新选择脚本。

**Q: 如何调试脚本？**

A: 使用固定的参数值测试几何生成，确认基础功能正确后再添加参数读取。

**Q: 支持哪些C语言特性？**

A: TCC支持大部分C99标准，但不支持某些高级特性。建议使用简单的控制流和基本数据类型。

**Q: 如何共享脚本？**

A: 将.toon文件复制到`scripts/`目录，重启程序或通过脚本库重新加载即可。
